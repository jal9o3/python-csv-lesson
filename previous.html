⚠️ The Problem with Python Lists

✅ Fast at

  • Index access: my_list[2]
  • Append to end

❌ Slow at

  • Inserting in the middle
  • Deleting from the middle

When you insert or delete in the middle, Python must shift all elements in memory.

🔗 Enter: Linked Lists


A Linked List is a collection of nodes connected by references.

Each node stores:

  • Some data
  • A reference to the next node

The chain:

Each element points to the next one in line.

🖼️ Visual Concept

[10 | next] ──▶ [20 | next] ──▶ [30 | None]

Each box = Node

Left side: the data

Right side: pointer to next node

End of list

The last node's next points to None.

🧱 Node Structure

A node is just an object: 

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None
self.data — stores the value
self.next — reference to the next node (None by default)

🔨 Building a Linked List

head   = Node(1)
second = Node(2)
third  = Node(3)

head.next   = second
second.next = third
head is a reference to the first node — our entry point into the list. 

head
 │
[1|next] ──▶ [2|next] ──▶ [3|None]

🚶 Traversing the List

current = head

while current:
    print(current.data)
    current = current.next
Start at head, visit each node, then move to current.next until we reach None.

📤 Output

1
2
3

Traversal = visiting each node one by one, from head to the end.

When current becomes None, the loop stops.

➕ Inserting at the Beginning

new_node = Node(0)
new_node.next = head
head = new_node
  1. Create the new node
  2. Point its next to the old head
  3. Update head to be the new node

➕ Inserting at the End

current = head

while current.next:
    current = current.next

current.next = Node(4)
Traverse to the last node (where current.next is None), then attach the new node.

🗑️ Deleting a Node

current = head

while current.next:
    if current.next.data == target:
        current.next = current.next.next
        break
    current = current.next
To delete a node: make the node before it skip over it — point directly to the node after it.

✅ Why Linked Lists?

Advantages over Python lists:

📏 Dynamic Size

Grows and shrinks without preallocating memory.

⚡ Fast Insert / Delete

No shifting elements — just update pointers.

🔁 Flexible Structure

Can be extended into stacks, queues, and more.

⚠️ Downsides

🚫 No Direct Indexing

Can't do my_list[3] — must traverse from head.

💾 More Memory

Each node stores an extra pointer (next).

🐢 Slower Traversal

Must visit each node one by one to find elements.

📊 Comparison

Feature Python List Linked List
Index access ✅ Fast ❌ Slow
Insert / Delete (middle) ❌ Slow ✅ Fast
Memory use ✅ Efficient ⚠️ Extra (pointers)
Dynamic size ✅ Yes ✅ Yes
