💡 What is Object-Oriented Programming?

OOP is a programming paradigm based on objects.


📦 Data (Attributes)

Information an object stores about itself.

Example: a dog's name, age, breed

⚙️ Behavior (Methods)

Actions an object can perform.

Example: bark, run, fetch


OOP helps organize code into reusable, modular structures.

🚗 Real World Analogy

Think about a Car 🚗

Characteristics (Attributes)

  • Color
  • Brand
  • Speed

Behaviors (Methods)

  • start()
  • stop()
  • accelerate()

Class = Blueprint   |   Object = Actual Car

📐 What is a Class?

A class is a blueprint used to create objects.


Class: Dog

Possible attributes:

  • name
  • age

Possible behaviors:

  • bark()
  • sleep()

A class is just the definition. Objects are the actual things created from it.

🐍 Python Class Syntax

class Dog:
    pass

class

Keyword that defines a class

Dog

Class name — capitalize by convention (PascalCase)

pass

Placeholder — class body is empty for now

🔨 Creating an Object

class Dog:
    pass

my_dog = Dog()
print(type(my_dog))   # <class '__main__.Dog'>
my_dog is now an object (instance) created from the class Dog.

Each call to Dog() creates a brand new, independent object.

🗃️ Attributes

Attributes store data about an object. 

class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age
self refers to the current object being created. Each object gets its own copy of the attributes.

🔑 What is __init__?

__init__ is a constructor method. It runs automatically when a new object is created. 

dog1 = Dog("Buddy", 3)
# __init__ runs here automatically
# sets dog1.name = "Buddy", dog1.age = 3
Think of __init__ as setup instructions that run every time a new object is born.

🔍 Accessing Attributes

dog1 = Dog("Buddy", 3)

print(dog1.name)   # Buddy
print(dog1.age)    # 3
Use dot notation: object.attribute

Read an attribute

print(dog1.name)

Change an attribute

dog1.name = "Rex"

⚙️ Methods

Methods define the behavior of objects. 

class Dog:

    def __init__(self, name):
        self.name = name

    def bark(self):
        print(self.name + " says Woof!")
Every method takes self as its first parameter — this lets it access the object's own data.

▶️ Using Methods

dog1 = Dog("Buddy")
dog1.bark()   # Buddy says Woof!
Use dot notation to call methods: object.method()

Without arguments

dog1.bark()

With arguments

dog1.eat("bone")

🐾 Multiple Objects

One class can create many objects. 

dog1 = Dog("Buddy")
dog2 = Dog("Max")
dog3 = Dog("Bella")

dog1.bark()   # Buddy says Woof!
dog2.bark()   # Max says Woof!
dog3.bark()   # Bella says Woof!
Each object has its own data but shares the same class blueprint.

🗂️ Object State

All created from the same class blueprint:

Object name
dog1 Buddy
dog2 Max
dog3 Bella
Each object is independent — changing dog1.name does not affect dog2.name.

➕ Adding More Methods

class Dog:

    def __init__(self, name):
        self.name = name

    def bark(self):
        print("Woof!")

    def greet(self):
        print("Hello, I am " + self.name)

dog1 = Dog("Buddy")
dog1.bark()    # Woof!
dog1.greet()   # Hello, I am Buddy
